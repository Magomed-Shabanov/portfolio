# Инструкция по исправлению ошибки DEPLOYMENT_NOT_FOUND

## Проблема
Ваш проект не связан с Vercel, поэтому при попытке доступа к деплойменту Vercel не может его найти.

## Решение (пошагово)

### Шаг 1: Вход в Vercel

Выполните в терминале:
```powershell
vercel login
```

**Что произойдет:**
- Откроется браузер
- Войдите в свой аккаунт Vercel (или создайте новый на vercel.com)
- После входа терминал покажет успешное сообщение

**Если браузер не открылся:**
- Скопируйте ссылку из терминала
- Вставьте её в браузер вручную
- Войдите в аккаунт

### Шаг 2: Связывание проекта с Vercel

После успешного входа выполните:
```powershell
vercel link
```

**Вам будут заданы вопросы:**
1. **Set up and deploy?** → Нажмите `Y` (Да)
2. **Which scope?** → Выберите ваш аккаунт или команду
3. **Link to existing project?** → 
   - Если проект уже есть в Vercel: выберите `Y` и укажите название
   - Если проекта нет: выберите `N` и введите новое название

### Шаг 3: Деплой проекта

После связывания выполните:
```powershell
vercel --prod
```

Или для превью деплоймента:
```powershell
vercel
```

**Что произойдет:**
- Vercel соберет ваш Next.js проект
- Загрузит его на серверы Vercel
- Дам вам URL для доступа к сайту

### Шаг 4: Проверка

После деплоя:
1. Откройте https://vercel.com/dashboard
2. Найдите ваш проект
3. Убедитесь, что деплоймент имеет статус "Ready" (Готов)
4. Откройте URL вашего сайта

## Альтернативный способ (через Git - рекомендуется)

Если вы хотите автоматические деплойменты при каждом push:

### 1. Инициализируйте Git (если еще не сделано):
```powershell
git init
git add .
git commit -m "Начальный коммит"
```

### 2. Создайте репозиторий на GitHub:
- Зайдите на github.com
- Создайте новый репозиторий
- Скопируйте команды для push (они будут показаны на GitHub)

### 3. Загрузите код на GitHub:
```powershell
git remote add origin <ваш-URL-репозитория>
git branch -M main
git push -u origin main
```

### 4. Подключите в Vercel:
1. Зайдите на https://vercel.com/new
2. Нажмите "Import Git Repository"
3. Выберите ваш репозиторий
4. Нажмите "Deploy"

**Преимущества этого способа:**
- ✅ Автоматические деплойменты при каждом push
- ✅ Превью деплойменты для каждой ветки и PR
- ✅ История всех деплойментов
- ✅ Легкий откат при проблемах

## Что делать, если что-то пошло не так?

### Ошибка: "Build failed"
- Проверьте, работает ли `npm run build` локально
- Исправьте ошибки сборки
- Убедитесь, что все зависимости установлены

### Ошибка: "Project not found"
- Убедитесь, что вы вошли в правильный аккаунт Vercel
- Проверьте права доступа к проекту

### Ошибка: "Deployment not found"
- Убедитесь, что деплоймент завершился успешно (статус "Ready")
- Проверьте правильность URL деплоймента
- Убедитесь, что проект правильно связан (есть папка `.vercel`)

```powershell
# Проверка установки Vercel CLI
vercel --version

# Проверка входа
vercel whoami

# Проверка связи проекта
Test-Path .vercel

# Проверка сборки
npm run build
```
